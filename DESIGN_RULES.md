# 1212 HP Design Rules

このドキュメントは `1212____HP` のUI/ビジュアルを一貫させるためのルールです。
新規ページ追加、既存ページ改修、管理画面（`admin/`）の改修時も、原則ここに従います。

## 0. 1212 Design System（音楽活動用）

このサイトは、`1212 Design System` の **music activity** トークンを利用します。

- Tokens (CSS): `assets/css/1212-music.tokens.css`

## 1. 目的

- **一貫性**: ページ間・セクション間で「同じサイト」に見えること
- **制約で統一**: モノクロを基本に、アクセントを **フィエスタレッド `#BF554D` 1色**に絞る
- **DIY x 洗練**: 余白・整列・最小限の装飾で、手触り感は残しつつ整える

## 2. カラー

### 2-1. 基本方針

- **メインカラー**: 黒 / 白 / グレーのみ
- **アクセントカラー**: フィエスタレッド **`#BF554D` のみ**
- フィエスタレッドの使用量は **全体の 10-20% 以下**（理想は 5-10%）
- 追加の「赤茶」「オレンジ寄り」など、**別色の増殖は禁止**

### 2-2. トークン（推奨）

CSSでは、原則として `assets/css/style.css` の `:root` 変数（デザイントークン）を使い、
ハードコードの色指定を避けます。

- `--paper`: `#ffffff`（背景）
- `--paper-deep`: `#f5f5f5`（淡いグレー背景）
- `--ink`: `#1a1a1a`（本文/見出しの標準テキスト）
- `--ink-muted`: `#666666`（補助テキスト）
- `--line`: `rgba(0, 0, 0, 0.12)`（罫線/境界）
- `--accent`: `#BF554D`（アクセント）
- `--accent-soft`: `rgba(191, 85, 77, 0.12-0.2)`（アクセント背景: alphaで調整）

補足:
- 追加のアクセント色（別HEX）の増殖はしない。濃淡は **不透明度** と影/枠線で表現する。

## 3. タイポグラフィ

### 3-1. 方針

- **サンセリフ（細字〜レギュラー）を基本**にする
- 日本語はゴシック系を優先し、**字間/行間をやや広め**に取る
- 文章は短く、長文が必要な場合も「段落を短く」して読みやすくする

### 3-2. ルール

- 見出し、本文、UIラベルのフォントファミリーは原則統一（ページごとに変えない）
- フォントウェイトは 2-3段階まで（例: 300/400/500）
- 見出しのサイズ差はやりすぎず、**余白で階層**を作る

## 4. レイアウト / 余白

### 4-1. 余白の基本

- 「詰めない」。余白は世界観の一部として扱う
- セクション間は一定のリズムで揃える（例: 56-80px）

### 4-2. スペーシングスケール（推奨）

余白・gap・paddingは以下のスケールから選ぶ（中途半端な数値を増やさない）。

- `4, 8, 12, 16, 24, 32, 48, 64, 80`

## 5. UIコンポーネント

### 5-1. リンク

- 通常は本文色（モノクロ）で表示し、hover/activeでフィエスタレッドを使う
- 常時フィエスタレッド色のリンクは乱用しない（重要導線のみ）

### 5-2. ボタン

- Primary: フィエスタレッド背景 + 白文字（ただし乱用しない）
- Secondary: モノクロ（白地/黒枠 or 黒地/白枠）
- 状態（hover/focus/disabled）は **彩度の違いではなく**、明度・不透明度・枠線・影で表現する

### 5-3. カード / 枠 / 影

- 枠線は基本 `--mono-20`（薄いグレー）
- 影は控えめにして、サイト全体で「同じ影」を使い回す
- 角丸は2-3段階までに制限する（例: 10/16/24）

### 5-4. ステータス（成功/警告/エラー）

原則モノクロで表現し、必要ならフィエスタレッドを添える（例: 左ボーダー、アイコン、下線）。
強い赤/緑のベタ塗りのバナーは避ける。

## 6. 写真 / ビジュアル

- 写真は基本 **モノクロ**（または彩度を極端に落とす）
- フィエスタレッドは「文字」「細いライン」「小さな図形」など **1要素だけ**に限定する
- 背景がうるさい写真は避け、ネガティブスペースを確保する

## 7. 実装ルール（重要）

- 色は **デザイントークン**を使う（新しい色の直書きを増やさない）
- フォント/余白/角丸/影も、できるだけトークン（変数）で管理する
- 迷ったら「装飾を足す」より「削る」「余白を増やす」

## 8. チェックリスト（PR前セルフレビュー）

- フィエスタレッドが増えすぎていないか（10-20%以内）
- グレーの段階が増殖していないか（2-3段階に収まっているか）
- ボタン/リンクの見た目がページ間で同じか
- 余白のスケールが揃っているか（4/8/12/16/24/32...）
- フォントがページごとに変わっていないか
